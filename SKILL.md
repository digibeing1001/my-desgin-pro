基于多模型AI生图（Seedream/Nano Banana/混元/GPT Image/Imagen 4/Flux 2/Ideogram/Stable Image） + canvas-design设计哲学方法论 + brand-cog品牌战略方法论 + design-critique设计评审 + svg-draw矢量绘制 + 国内合规审查的专业平面设计工作流。

## 🚀 自动触发规则（R-TRIGGER）

> **只要意识到用户的意图涉及品牌/视觉/设计产出，就必须立即加载本Skill。宁可误触不可漏触。**

| # | 规则 | 违反后果 |
|---|------|---------|
| R-T1 | **意图识别触发**：用户表达任何"做设计/出图/画/视觉/品牌"意图时，立即 `use_skill graphic-design-pro`，不用等用户说出精确术语 | 漏触发 = 用户得不到专业流程 |

| R-T3 | **子任务也触发**：如果用户的大任务中包含任何设计子环节（如"做个产品页面"→包含Banner设计），设计环节必须用本Skill | 跳过 = 设计降质为非专业输出 |
| R-T4 | **合规审查独立触发**：用户只要求"审查这个设计/看看合不合规"时，也触发本Skill（进入Phase 5） | 不触发 = 合规审查缺失 |
| R-T5 | **不与纯代码UI冲突**：用户要求"写一个网页/App界面"（纯代码实现）→ 用 ui-design-review / frontend-design；但若涉及品牌视觉产出（Logo/品牌色/VI规范），必须同时触发本Skill | 只用UI Skill = 品牌一致性缺失 |

## 🎯 模式路由（项目复杂度分级）

> **不在简单项目上做重型流程。** 根据项目规模选择合适的深度。

| 模式 | 何时用 | 做法 |
|------|--------|------|
| 🟢 **轻量级** | 单一物料（一张海报/一个Logo） | Phase 1 快速追问(3轮) → Phase 3 直接出图 → Phase 5 合规检查 |
| 🟡 **中等** | 小型品牌套件（Logo+名片+1-2物料） | Phase 1 → Phase 2(压缩) → Phase 3 → Phase 4(精选物料) → Phase 5 |
| 🔵 **深度** | 完整VI系统（多物料+多场景） | 完整6阶段，每步充分展开 |
| 🔴 **重构** | 已有品牌升级/重塑 | 深度模式 + 额外回溯：旧品牌资产盘点、迁移策略、变更影响 |

**路由判断流程：**
```
收到设计需求
    ↓
是单一物料？
    ↓ 是          ↓ 否
轻量级        是全新品牌还是升级？
                 ↓ 全新          ↓ 升级
              物料<5个？       重构模式
              ↓ 是    ↓ 否
            中等模式  深度模式
```

## ⚠️ Top 5 翻车场景（每Phase开始前快速扫描）

| # | 翻车模式 | 表现 | 怎么防 |
|---|---------|------|--------|
| 1 | **品牌名漂移** | 不同产出用了不同品牌名 | R1逐字锁定 + brand-profile.md |
| 2 | **需求理解偏差** | 做完了才发现不是用户想要的 | Phase 1 Clarify追问法（见下）+ R8逐步确认 |
| 3 | **风格分裂** | 不同物料看起来不像同一个品牌 | R3-R7锁定 + 审美档案 + 辅助图形贯穿 |
| 4 | **合规后置** | 做完才发现违法 | R9合规前置 + Phase 5双重审查 |
| 5 | **过度设计** | 物料太多、流程太长 | 奥卡姆剃刀 + 模式路由 + 二八法则 |

## ⛓️ 品牌一致性铁律（全流程强制执行）

> **这不是建议，是硬规则。违反 = 产出作废。**

品牌名确认后，**立即**在项目目录下创建 `brand-profile.md`，写入锁定项。

### 硬规则

| # | 规则 | 违反后果 |
|---|------|---------|
| R1 | **品牌名逐字锁定**：后续所有产出必须逐字引用 brand-profile.md，不得擅自替换、缩写、意译或混用 | 产出作废，重做 |
| R2 | **禁止自作主张改名**：不得将用户确认的品牌名"优化"为自认为更好的名字 | 严重违规 |
| R3 | **色彩值锁定**：所有物料必须使用 brand-profile.md 中的 HEX 值 | 产出作废 |
| R4 | **字体锁定**：标题/正文字体确定后，所有物料统一使用 | 产出作废 |
| R5 | **设计哲学锁定**：Phase 2 的设计哲学是全流程创作纲领，Phase 3-6 不得偏离 | 产出作废 |
| R6 | **每阶段开头读 brand-profile.md**：进入新 Phase 时必须先读取确认 | 跳过 = 违规 |
| R7 | **品牌口号锁定**：不得在不同物料中使用不同口号 | 产出作废 |
| R8 | **逐步确认，禁止跑通流程**：每个Phase/每个核心物料完成后，必须暂停等待用户确认 | 跳过确认 = 产出无效 |
| R9 | **合规审查前置**：出品每个物料前，先扫描 `references/design-compliance.md` 内化到prompt | 做完再查 = 违规产出 |
| R10 | **包装物料双产出规则**：实体包装必须同时提供①**结构刀图/展开图**（生产用）+②**成品效果图/Mockup**（展示用），两者缺一不可 | 只有刀图 = 缺少视觉确认；只有效果图 = 无法落地生产 |
| R11 | **素材预渲染复用规则（R-ASSET）**：Logo/品牌文字/辅助图形等跨物料重复使用的元素，必须在首次确认后**立即预渲染为高清PNG素材文件**保存到项目 `assets/` 目录；后续所有物料（包装/网站/海报/名片等）直接引用这些素材文件，不得重新绘制或让AI生图生成 | 每次重绘 = 结构漂移、比例偏差、颜色不一致；违反 = 产出作废 |

### 一致性自检（每个物料产出前必过）

```
✅ 品牌名是否与 brand-profile.md 逐字一致？
✅ 色彩是否使用 brand-profile.md 中的 HEX 值？
✅ 字体是否与 brand-profile.md 一致？
✅ 设计哲学是否与 Phase 2 一致？
✅ 品牌口号是否与 brand-profile.md 一致？
✅ 视觉风格（圆角/锐角、留白比例、辅助图形）是否与已确认方向一致？
✅ 中文字号是否≥英文字号？（招牌/广告/包装等有中文+外文并存的物料）
✅ 合规硬约束是否已前置扫描并内化到prompt/设计参数中？
✅ 包装类物料是否同时提供刀图+成品效果图（R10双产出）？
✅ 跨物料重复元素（Logo/品牌名/辅助图形）是否使用预渲染素材而非重绘（R11）？
✅ 食品包装是否预留了强制标签区域？
✅ 色彩是否标注了CMYK和Pantone值（印刷物料）？
```

### 输出前5问（融入PM-Clarity自检）

> 每次交付Phase产出前扫描，**任一答案为"否"则必须修正**：

1. ❓ 我理解了用户的真实设计需求吗？（不是表面需求）
2. ❓ 我暴露了关键设计假设吗？（区分了"必须"和"惯例"）
3. ❓ 我在不删除品牌一致性的前提下精简了吗？（没有因为偷懒省略必要物料）
4. ❓ 产出以可执行决策/行动收尾了吗？（不是纯分析或展示）
5. ❓ 用户知道下一步做什么吗？（有明确确认点和后续安排）

### 🛑 追问停止规则（融入my-skill）

**规则A：用户主动喊停** — 用户说"可以了""够了""就这样""没问题" → 立即停止追问，进入产出
**规则B：精确度达标** — Phase 1追问精确度连续2次≥2分（轻量级）或≥3分（深度模式） → 可推进
**规则C：最多轮到达** — 轻量级3轮、中等5轮、深度8轮 → 到达上限先输出【需求简报】，再问"还差什么没聊到？"

**需求简报模板：**

```
✅ 已确认：
- 品牌名/项目名：xxx
- 核心设计目标：xxx
- 目标受众：xxx
- 调性关键词：xxx

❓ 可能还缺的：
- xxx

还差什么没聊到？
```

### 🔬 Phase 1 Clarify追问法（融入PM-Clarity）

> **不要把错误的需求设计得很漂亮。** Phase 1 追问不只是"问清楚"，要像PM一样追问到不可再分解。

**① 重述表面需求**：用结果语言重述，不只复述用户的框架

| 用户说 | 可能的真实需求 |
|--------|--------------|
| 「我要一个高端Logo」 | 「提升品牌信任感/权威性」 |
| 「设计要大气」 | 「目标场景是什么？户外招牌要远距离可辨？还是名片上要精致？」 |
| 「想要苹果那种风格」 | 「极简+大量留白+无衬线字体+产品为中心」 |

**② 识别模糊用词**：设计领域的常见模糊词：

| 模糊词 | 追问方式 |
|--------|---------|
| 大气 | 大气给谁看？什么让它看起来大气？是留白多？还是字大？ |
| 高端 | 高端的衡量标准是什么？材质？色彩？排版？ |
| 年轻 | 什么年龄段的年轻？二次元年轻还是潮流年轻？ |
| 专业 | 专业给同行看还是给客户看？什么让它看起来专业？ |
| 简约 | 简约到什么程度？极简留白还是信息精简？ |

**③ 区分目标与手段**：
- **手段伪装成目标**：「我要一个3D Logo」→ 真实目标可能是「让品牌看起来有科技感」
- **恐惧伪装成目标**：「颜色不能太素」→ 真实恐惧可能是「怕和竞品混淆」
- **跟风伪装成需求**：「别人都有品牌IP形象」→ 真实需求是什么？IP形象服务于什么？

**④ 暴露初始假设**：这个设计需求成立的前提条件是什么？
- 用户对目标受众的描述有数据支撑吗？
- 「我们的品牌调性是XX」——是经过分析的还是拍脑袋的？
- 对竞品的判断来自观察还是想象？

**⑤ 需求精确度评分**（每轮追问后自评）：

| 分数 | 含义 | 策略 |
|------|------|------|
| 0分 | 模糊、套话（"大气""高端"） | 必须追问，不推进 |
| 1分 | 有方向但粗糙 | +1轮追问补细节 |
| 2分 | 具体但缺边界 | +1轮追问补边界 |
| 3分 | 完整清晰（有受众/场景/调性/参考） | 进入下一阶段 |

> **分数 <2 时，该阶段不计入有效追问轮数。** 轻量级≥2分即可推进，深度模式需≥3分。

**⑥ 重写需求**：给出最锐利版本的真实设计需求

### 🔍 设计决策中的假设管理

> **设计决策不能把假设当事实。** 区分硬约束和软约束。

| 类型 | 定义 | 设计中的例子 |
|------|------|-------------|
| **硬约束** | 不可违反（法规/物理/已锁定品牌） | 《商标法》§10禁用标志、印刷出血3mm、品牌色HEX值 |
| **软约束** | 可挑战的惯例 | 「Logo必须圆形」「行业都用蓝色」「标题必须加粗」 |

> **绝不在没有论证的情况下将软约束呈现为不可变的。** 发现软约束时标注：「这是惯例，不是铁律。如果你愿意挑战，我们可以讨论替代方案。」

**设计假设暴露清单（Phase 2/3产出前自检）：**
- 什么被当成了"必须的"？（如"必须有渐变"）
- 哪个设计选择承载了最大权重？（如Logo的核心识别点）
- 哪个假设如果不成立，设计需要大改？（如假设受众是年轻人→实际是中年人）

### 🔄 Reflect 7步法（融入my-skill）

当用户纠正或设计产出被否定时，自动进入反思流程：

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1. Identify | 定位根因 | 哪个Phase出了问题？是指令缺失/有歧义/参考过时/描述太窄？ |
| 2. Read | 重读原文 | 重读相关SKILL.md和phase-N.md，确认gap存在（别靠记忆） |
| 3. Impact scan | 跨文件影响 | 搜索其他phase-N.md和references/是否引用同一概念 |
| 4. Determinism | 确定性选择 | 能写成固定规则的→改SKILL.md/phase-N；需要LLM判断的→保留指令+加示例 |
| 5. Propose | 提出方案 | 改哪个文件+加什么/删什么/改什么+为什么能防止复发 |
| 6. Confirm | 等待同意 | 向用户展示方案，等明确同意后再改 |
| 7. Apply | 执行修改 | 修改文件 + 升级版本号 + 写进化日志 + 写CHANGELOG.md |

**升级警告：**
- 同一问题连续反思2次 → 停止打补丁，考虑根本方案是否有误
- 不同问题累计3+次反思 → 结构性问题，考虑是否需要拆分Phase或重组规则

### 🔁 知识库自动扫描规则（R-KB）

> **机制**：design-references.md 头部包含 `🕐 最后扫描时间` 时间戳
>
> **触发逻辑**：
> 1. **Skill首次启动** → 检查时间戳 → 如果不存在或距当前时间 > **14天** → 触发全站扫描
> 2. **用户频繁使用**（14天内使用过）→ 每 **14天** 自动扫描一次
> 3. **用户长期未用**（距上次使用 > 14天）→ 下次启动时 **立即扫描**
>
> **扫描内容**：design-references.md §八-B 中标记为 `—`（未扫描状态）的网站
> - 使用 web_fetch 工具逐站抓取首页/核心页面

> - 追加到对应章节，更新该站的"扫描状态"
> - 扫描完成后更新 `🕐 最后扫描时间` 为当前ISO时间
>
> **时间戳格式**：`YYYY-MM-DDTHH:MM:SS` (ISO 8601)

## 关键原则

1. **先追问，后生成** — 不急于出图，先确保需求清晰（Phase 1 Clarify追问法）
2. **品牌一致性铁律** — R1-R10一旦确认，全流程锁死
3. **品牌战略先行** — 品牌类设计先定义定位和个性，再谈视觉
4. **竞品分析必做** — 品牌类设计必须先分析竞品视觉
5. **设计哲学驱动** — 每套设计都有统一的美学纲领
6. **样稿确认再扩展** — 避免方向错误导致大量返工
7. **合规内置** — 生成时即扫描（R9），不是事后补审
8. **版权红线** — 字体版权、商标法§10、AI著作权风险，零容忍
9. **风格统一** — 所有物料共享同一视觉基因
10. **审美自检** — 每个产出都要通过审美质量自检清单
11. **辅助图形不缺席** — 辅助图形是VI基础系统的必要组件
12. **色彩有比例** — 主色60%+辅助色30%+强调色10%
13. **落地闭环** — 必须提供印刷/数字检查清单和VI维护指南
14. **输出实用** — 矢量/PSD可编辑，不是只能看的位图
15. **工艺极致** — 每件作品都应看起来经过无数小时精心制作
16. **禁止自由发挥** — 品牌确认的内容是铁律，不是"参考"
17. **解释为什么，不堆砌MUST** — 给理由比下命令更有效，LLM有推理能力（融入my-skill）
18. **假设必须暴露** — 设计决策中的"惯例"必须标注可挑战，不能把假设当事实（融入PM-Clarity）
19. **模式路由** — 不在简单项目上做重型流程，按复杂度选择深度（融入PM-Clarity）
20. **复利积累** — 每个项目的审美反馈都让下一个项目更好（融入my-skill）
