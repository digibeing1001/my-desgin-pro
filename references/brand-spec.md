> 本文件为「核心资产协议」的固化输出，是品牌视觉资产的单一真实来源（Single Source of Truth）。

## 采集流程（Step 1-4）

### Step 1 · 问（资产清单一次问全）

不要只问「有 brand guidelines 吗？」——太宽泛。按清单逐项问：

```
关于 <brand/product>，你手上有以下哪些资料？我按优先级列：
1. Logo（SVG / 高清 PNG / JPG）—— 任何品牌必备
2. 产品图 / 官方渲染图 —— 实体产品必备
3. UI 截图 / 界面素材 —— 数字产品必备
4. 色值清单（HEX / RGB / 品牌色盘）
5. 字体清单（Display / Body）
6. Brand guidelines PDF / Figma / 品牌官网链接

有的直接发我，没有的我去搜/抓/生成。
```

**用户提供素材的强制处理（R15）**：
- 用户上传任何文件 → **立即保存**到 `assets/user-uploads/`，文件名保留原始名称+时间戳（如 `logo-20260422-163000.jpg`）
- **禁止**让用户提供的素材停留在对话历史中而不保存到文件系统
- 在 `brand-profile.md` 中新增「用户资产」字段记录路径
- 所有后续产出（生图 prompt / HTML / Canvas）必须引用该路径，禁止重绘或忽略

### Step 2 · 搜官方渠道（按资产类型）

| 资产 | 搜索路径 |
|---|---|
| **Logo** | `<brand>.com/brand` · `<brand>.com/press` · `<brand>.com/press-kit` · 官网 header 的 inline SVG |
| **产品图/渲染图** | `<brand>.com/<product>` 产品详情页 hero image + gallery · 官方新闻稿附图 |
| **UI 截图** | App Store / Google Play 产品页截图 · 官网 screenshots section |
| **色值** | 官网 inline CSS / brand guidelines PDF |
| **字体** | 官网 `<link rel="stylesheet">` 引用 · brand guidelines |

### Step 3 · 下载资产 · 三条兜底路径

**3.1 Logo（任何品牌必需）**
- 独立 SVG/PNG 文件（最理想）
- 官网 HTML 全文提取 inline SVG（80% 场景必用）
- 官方社交媒体 avatar（最后手段）

**3.2 产品图/渲染图（实体产品必需）**
- 官方产品页 hero image（最高优先级，通常 2000px+）
- 官方 press kit
- 官方 launch video 截帧
- **AI 生成兜底**（以官方参考图为基底生成符合场景的变体）
- **禁止用 CSS/SVG 手画代替**

**3.3 UI 截图（数字产品必需）**
- App Store / Google Play 产品截图
- 官网 screenshots section
- 产品演示视频截帧

### Step 4 · 验证 + 提取

| 资产 | 验证动作 |
|---|---|
| **Logo** | 文件存在 + 可打开 + 至少两个版本（深底/浅底）+ 透明背景 |
| **产品图** | 至少一张 2000px+ 分辨率 + 去背或干净背景 + 多个角度 |
| **UI 截图** | 分辨率真实 + 是最新版本 + 无用户数据污染 |
| **色值** | 从真实 HTML/SVG 抽取，过滤黑白灰，警惕示范品牌污染 |

**警惕示范品牌污染**：产品截图里常有用户 demo 的品牌色，那不是该品牌色。**同时出现两种强色时必须区分**。

**品牌多切面**：同一品牌的官网营销色和产品 UI 色经常不同。两套都是真的——根据交付场景选合适的切面。

## 7. 素材使用黑名单（Anti-AI-Slop）

| 禁止项 | 为什么禁止 | 何时允许 |
|--------|-----------|---------|
| CSS/SVG 产品剪影 | 假的产品轮廓 = 假的品牌信任 | 永远禁止，用真实产品图 |
| 过度紫色渐变背景 | AI 生成图的廉价标志 | 品牌色包含紫色时可用 |
| 无意义的几何装饰 | 与品牌内容无关联 = slop | 从 Logo 提取的辅助图形可用 |
| Emoji 替代专业图标 | 不专业，破坏品牌感 | 社交媒体 casual 内容可用 |
| 假数据/假图表 | 误导受众 | 使用真实数据或标注「示例数据」 |
| 生成式 AI 人物照片 | 恐怖谷/版权风险 | 明确标注 AI 生成 + 非真人场景 |
| 通用库存图（握手/攀登/迷宫） | 无品牌辨识度 | 经品牌化处理后可用 |
