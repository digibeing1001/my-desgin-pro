2. 确认 Phase 3 已通过 P4-G3

## Layer 1 · 表面层（物料清单确认 + 色彩系统）

> **目标：确认「做什么物料」，不开始做任何设计。**
> **禁止在 Layer 1 生成任何物料。**

### 此刻只做 2 件事：

1. **推荐物料清单**（根据行业和场景，给出推荐列表）
2. **生成色彩系统**（使用 `scripts/color_system.py`）

### 物料清单推荐格式

```
推荐物料清单
━━━━━━━━━━━━━━━━━━━
🔴 必须（建议全部保留）：
• {物料A} · {说明}
• {物料B} · {说明}

🟡 推荐（根据预算选择）：
• {物料C} · {说明}
• {物料D} · {说明}

🟢 可选（大型项目再考虑）：
• {物料E} · {说明}
```

### 色彩系统生成

```bash
python scripts/color_system.py "{主色HEX}" --output brand-colors.json
```

**只生成色彩系统文件，不开始设计物料。**

### Layer 1 执行规则

- **不给用户 overwhelm**：推荐 ≤8 个物料（必须+推荐）
- 用户可增减，确认后再开始设计
- 包装类物料在清单中标注「含刀图+效果图（R10）」

### P4-G1 · 方向确认门

```
📍 Phase 4 · Layer 1 完成
━━━━━━━━━━━━━━━━━━━
推荐物料：{N}项
色彩系统：已生成（{brand-colors.json 路径}）

请确认：
1. 物料清单是否需要调整？（保留 / 增减）
2. 色彩系统是否符合预期？（是 / 需调整）
3. 是否开始逐个设计物料？（是 / 否）
```

**用户未确认清单 → 禁止进入 Layer 2。**

## Layer 2 · 细节层（逐物料生成 + 逐个确认）

> **目标：基于确认的清单，逐个设计物料，每做一个确认一个。**
> **仅在用户通过 P4-G1 后解锁。**

### 2.1 执行策略

**逐个物料生成，不是批量生成。**

```
物料 1/8：名片
→ 生成 1-2 稿
→ 用户确认
→ 通过 → 下一个物料
→ 不通过 → 修改后再确认

物料 2/8：信纸
→ ...
```

### 2.2 每个物料的生成流程

**每个物料开工前必过「位置四问」**（融入huashu-design）：见 SKILL.md §位置四问。

1. **合规前置扫描（R9）**：生成前扫描 `references/design-compliance.md` 中该物料的关键约束（只扫描最相关的 3-5 项，不是全部 20 项）
2. **构造 Prompt**：基于 brand-profile.md + 设计哲学 + 预渲染素材（R11）
3. **素材质量门（5-10-2-8）**：如需参考图/产品图/背景素材，执行 5-10-2-8 门槛。见 SKILL.md §5-10-2-8。不够 8 分宁可不用，改用 honest placeholder。
4. **生成 1-2 稿**
4. **一致性自检**：过 R1-R11 清单（只过与当前物料相关的项）
5. **一致性自检**：过 R1-R11 清单（只过与当前物料相关的项）
6. **展示并等待确认**

### 2.3 多物料排版策略（R16 · 产出路径分叉）

**必须先确定产出路径，再选择工具：**

| 用户要的产出 | 技术路径 | 工具 | 说明 |
|-------------|---------|------|------|
| **精确排版稿**（可直接印刷/上线） | 路径 A：代码/Canvas 精确渲染 | HTML/CSS/Canvas-design/SVG | 像素级精确，AI 生图不走此路 |
| **AI 概念图/风格探索** | 路径 B：AI 生图模型创作 | Seedream/Flux/Imagen 等 | 基于 prompt 重新创作，不保证像素级复现 |
| **场景 Mockup**（通用样机：海报贴白墙/产品放桌面） | 路径 C-Mockup：精确稿 → 通用样机 | 精确稿导出 PNG → 样机模板/PS 智能对象 | 基于通用场景模板展示设计 |
| **真实场景融合**（前台/门头/展墙等实际空间） | 路径 C-Real：精确稿 + 真实照片融合 | 精确稿 PNG + 用户场景照片 → 图生图/图像编辑 | 设计稿融入用户提供的真实场景，保持原场景光照透视（R17） |

### 路径决策流程

```
用户要求「效果图」
    ↓
问：你要的是「精确设计稿」（可印刷/上线的成品）还是「场景 Mockup」（展示设计贴在墙上/产品上的样子）？
    ↓
精确设计稿 → 路径 A（HTML/Canvas/SVG）
    ↓ 导出 PNG/PDF 即为最终交付物，禁止让 AI 生图"重画"
    
通用场景 Mockup → 路径 C-Mockup
    ↓ Step 1: 先完成精确设计稿（路径 A）
    ↓ Step 2: 导出 PNG
    ↓ Step 3: 使用样机模板或图生图生成通用场景展示
       Prompt 示例："Product mockup, a poster with [设计内容描述] printed on it,
       displayed on a modern office wall, natural lighting, photorealistic"
    ↓ Step 4: 如需要，用 Photoshop/脚本将精确稿 PNG 直接贴入样机模板

真实场景融合（前台/门头/展墙）→ 路径 C-Real（R17）
    ↓ Step 1: 分析用户场景照片（光照/透视/材质/比例）
    ↓ Step 2: 估算尺寸 → 用户确认/修正真实尺寸
    ↓ Step 3: 按真实比例完成精确设计稿（路径 A），以场景照片为背景参照
    ↓ Step 4: 导出设计稿 PNG
    ↓ Step 5: 使用图生图/图像编辑将设计稿融入真实场景
       • 将用户场景照片作为 reference_image
       • 将设计稿 PNG 作为第二张参考图（如模型支持多参考图，如 Nano Banana 2 支持14张）
       • Prompt: "Place this [招牌/海报/logo墙] design onto the real wall in the reference photo,
         matching the exact perspective, lighting, and material of the scene. Photorealistic."
    ↓ Step 6: 检查结果：透视是否对齐？光照方向是否一致？材质是否融合自然？
    
风格探索 → 路径 B
    ↓ 与精确稿无关，独立生成概念图供方向参考
```

### ⚠️ 严禁行为（R16 + R17）

- **禁止**将 HTML/Canvas 精确稿截图后，作为「prompt 描述」让 AI 重新生成一张"相同的海报"。AI 会改变布局、字体、间距。
- **禁止**将 HTML 设计稿直接作为 reference_image 让 AI "优化"——AI 会将其视为风格参考，重新创作画面，不保留精确排版。
- **禁止脱离场景照片凭空设计"通用前台"后声称是"该位置的效果图"（R17）**——必须基于真实场景照片进行融合。
- **禁止用 AI 生图直接生成包装 3D 效果图（R18）**——AI 不理解包装盒的三维折叠结构，必然出现面连接错误、透视扭曲、不可能几何。包装效果图必须使用 3D 软件/专业包装工具/PSD 样机模板。
- **正确做法**：精确稿的产出就是 HTML/Canvas 本身，导出 PNG/PDF 即完成。如需更高分辨率，用浏览器截图工具（如 Playwright）而非 AI 生图。

| 物料类型 | 默认路径 | 例外 |
|---------|---------|------|
| 招牌/海报/易拉宝 | 路径 B（AI 概念图）或 路径 A（HTML 精确稿） | 用户已提供设计稿 → 直接走路径 C-Mockup |
| 名片/信纸/画册 | 路径 A（canvas-design 精确排版） | 需要氛围展示 → 路径 C-Mockup |
| **包装刀图** | **路径 A（SVG 精确绘制）** | **严禁用刀图直接当印刷文件，需出血/压痕/刀线标注** |
| **包装效果图** | **3D 软件/专业包装工具/Pacodora/PSD 样机模板** | **严禁用 AI 生图直接生成（R18），会导致空间扭曲** |
| Banner/网页头图/社交媒体图 | 路径 A（HTML/CSS 精确稿） | 需要场景展示 → 路径 C-Mockup |
| **前台背景墙/门头/展墙/办公室标识** | **路径 C-Real（真实场景融合）** | **必须基于用户场景照片，禁止通用模板** |

### 2.4 品牌规范手册框架

Phase 4 进行过程中，同步填充品牌规范手册框架（Markdown 版），为 Phase 6 的 PPT 做准备。

### Layer 2 执行规则

- **每次只做一个物料**，用户确认后再做下一个
- 使用预渲染素材（R11）：Logo/品牌名/辅助图形直接引用 `assets/` 中的 PNG，不重新生成
- 包装类物料必须同时提供刀图+效果图（R10）
- **包装效果图严禁使用 AI 生图直接生成**（R18 + `references/ai-image-limitations.md` §四），必须使用 3D 软件/专业工具/PSD 样机
- 每个物料生成时标注 CMYK + Pantone（印刷物料）

## 关键提示

- **R8 铁律**：每个物料逐个生成并经用户确认。禁止批量做完再展示。
- **R9 合规前置**：每个物料生成前即时预扫描合规，不是事后补审。
- **R10 包装双产出**：包装必须同时提供刀图+效果图，缺一不可。
- **R11 素材复用**：跨物料重复元素使用预渲染素材，不得重绘。
- 色彩系统使用比例：主色60% + 辅助色30% + 强调色10%。
