# 反 AI Slop 清单：平面设计审美陷阱速查

> 来源：融入 huashu-design 反 AI slop 方法论，适配平面设计场景。
> **这份清单比「做什么」更重要——因为 AI slop 是默认值，你不主动避免就会发生。**

---

## 什么是 AI Slop？

**AI Slop = AI 训练语料里最常见的「视觉最大公约数」**。

紫渐变、emoji 图标、圆角卡片+左 border accent、过度对称的构图——这些东西之所以是 slop，不是因为它们本身丑，而是因为**它们是 AI 默认模式下的产物，不携带任何品牌信息**。

**规避逻辑链**：
1. 用户请你做设计，是要**他的品牌被认出来**
2. AI 默认产出 = 训练语料的平均 = 所有品牌混合 = **没有任何品牌被认出来**
3. 所以 AI 默认产出 = 帮用户把品牌稀释成「又一个 AI 做的设计」
4. **反 slop 不是审美洁癖，是替用户保护品牌识别度**

> 服从 brand-spec.md 规范是反 slop 的**正向方式**（做对的事），这份清单是反 slop 的**反向方式**（不做错的事）。两者缺一不可。

## 一、视觉陷阱

### ❌ 激进渐变背景
- 紫色 → 粉色 → 蓝色 全屏渐变（AI 生成海报的典型味道）
- 任何方向的 rainbow gradient
- Mesh gradient 铺满背景
- ✅ **如果要用渐变**：subtle、单色系、有意图地点缀（比如按钮 hover 或局部高光），**不是铺满**

### ❌ 圆角卡片 + 左彩色 border accent
```
┌─────────────────────┐
│ █ 标题              │  ← 左 border 4px 彩色
│ 内容内容内容...     │
└─────────────────────┘
```
这种卡片在 AI 生成的宣传物料里泛滥。想做强调？用更有设计感的方式：
- 背景色对比（浅底上的深色块）
- 字重/字号对比
- 留白分隔
- 或者干脆**不分卡片**——用网格对齐即可

### ❌ Emoji 作装饰
- 标题前的 🚀 ⚡️ ✨ 🎯 💡
- Feature 列表的 ✅
- CTA 按钮里的 emoji 箭头

除非品牌本身使用 emoji（如 Notion、Slack），否则不要放。没图标用专业 icon 库，或用留白解决。

### ❌ AI 生成的「人脸/人物/手部」
AI 生成的人物永远有细微畸形（手指、眼睛对称、牙齿）。**平面设计中涉及人物肖像时**：
- 优先用真实照片（走核心资产协议）
- 次选用插画师作品或正版素材
- 必须用 AI 生成时，选择**背影/剪影/抽象化**处理，避开面部特写

### ❌ 过度对称构图
AI 默认倾向绝对对称（居中 + 两侧镜像），这会让设计显得**呆板、没有生命力**。
- ✅ 用**不对称平衡**替代绝对对称
- ✅ 主体偏左/偏右，用留白或次要元素在另一侧形成视觉平衡
- ✅ 网格系统中故意打破一格，创造节奏感

### ❌ 无意义的「科技感」装饰
- 渐变圆球代表 AI
- 数字雨/电路板纹理背景
- 蓝色发光线条环绕产品
- 机器人/芯片图标堆砌

**用户已经对这些视觉疲劳**，无法区分你的品牌和任何其他科技品牌。

## 二、字体陷阱

### ❌ 避免这些烂大街字体（作 display/标题时）

| 字体 | 为什么避免 | 替代方案（平面设计） |
|------|-----------|-------------------|
| Inter / Roboto | AI 生成网页默认 | Newsreader, Cormorant, Playfair Display |
| Arial / Helvetica | 无个性，系统默认 | Univers, Akzidenz-Grotesk（需授权） |
| 微软雅黑（纯排正文OK） | 当作 display 用 = 没风格 | 思源宋体、方正兰亭黑系列 |
| Fraunces（最近 AI 滥用） | 已烂大街 | Cormorant Garamond, Libre Baskerville |
| Space Grotesk | AI 新宠 | Bricolage Grotesque, Syne |

### ✅ 用有特点的 Display + Body 配对

**西文组合**：
- 衬线 display + 无衬线 body（editorial feel）→ Playfair Display + Lato
- Mono display + sans body（technical feel）→ JetBrains Mono + Open Sans
- Heavy display + light body（contrast）→ Oswald + Source Sans Pro

**中文组合**：
- 宋体 display + 黑体 body（书卷气）→ 思源宋体 + 思源黑体
- 书法 display + 黑体 body（国潮）→ 站酷高端黑 + 方正兰亭黑
- 圆体 display + 黑体 body（亲和）→ 站酷快乐体 + 思源黑体

**原则**：Display 字体负责「风格签名」，Body 字体负责「阅读舒适」。两者需有足够对比度（衬线vs无衬线、粗vs细、曲vs直）。

## 三、色彩陷阱

### ❌ 凭空发明颜色
不要从头设计一整套不熟悉的色彩。这通常不和谐。

**✅ 策略**：
1. **有品牌色** → 用品牌色，缺的 color token 用 oklch 插值
2. **没有品牌色但有参考** → 从参考图/竞品吸色
3. **完全从零** → 选一个 known 的配色系统（Pantone 年度色 / Adobe Color / Coolors），不要自己调

### ❌ 夜间模式随手加反色
平面设计通常不涉及 dark mode，但如果做数字物料（社交媒体图/网页配图）：
- 不是简单 invert 颜色
- 好的 dark mode 需要重新调整饱和度、对比度、accent 色
- 不想做就别做

### ❌ 多色聚类
- 海报上出现 5+ 种鲜艳色彩 = 马戏团效果
- 数据可视化除外（但即使是图表，也建议 ≤4 种区分色 + 灰阶）

**✅ 策略**：一个温暖/有温度的底色 + **单个** accent 贯穿全场。需要更多颜色时，用同色系深浅变化。

## 四、Layout 陷阱

### ❌ Bento grid 过度泛滥
AI 生成的 landing page、海报、社交媒体图都想搞 bento（等分小格子拼贴）。除非信息 structure 确实适合，否则用其他 layout。

**平面设计中的 bento 变体陷阱**：
- 名片正面分成 6 个小格 = 信息碎片
- 海报用 3×3 网格填满 = 没有焦点
- 画册每页都一样的 2×2 模块 = 模板感

### ❌ 大标题 + 3 列 feature + 引用 + CTA
这个海报/宣传单模板被用烂了：
```
[大标题]
[副标题]

┌───┐ ┌───┐ ┌───┐
│ 1 │ │ 2 │ │ 3 │   ← 3 个等宽 feature
└───┘ └───┘ └───┘

"用户评价引用"

[立即购买 CTA]
```

**这不是设计，是填空**。想要创新就真创新：
- 标题跨页出血
- Feature 用非对称排版
- 引用用大字报形式独占一屏
- CTA 融入图像而非单独区块

### ❌ 信息密度与载体不匹配

| 载体 | 常见错误 | 正确密度 |
|------|---------|---------|
| 海报 | 塞了 10+ 个元素 | 1 个视觉焦点 + ≤3 个辅助信息 |
| 名片 | 正面放地址/电话/邮箱/网址/二维码/标语 | 姓名 + 职位 + 品牌名 + 1 种联系方式 |
| 包装正面 | 成分表+使用方法+品牌故事全放 | 品牌名 + 产品名 + 1 个卖点 + 视觉焦点 |
| 画册封面 | 标题+副标题+简介+图片+装饰 | 标题 + 1 张主图 + 极简信息 |

## 五、内容陷阱

### ❌ "Data slop" — 编造的数据装饰
- "10,000+ happy customers"（你都不知道有没有）
- "99.9% uptime"（没有真数据就别写）
- 用图标+数字+词组成的装饰 "metric cards"

**平面设计中的变种**：
- 海报上的 "服务 500+ 企业"（没验证过）
- 包装上的 "销量第一"（没有来源）
- 宣传册上的虚假 statistics

**没真数据，留 placeholder 或问用户要**。

### ❌ "Quote slop" — 编造的用户评价
海报/宣传册上放 "某用户说：'这个产品改变了我的生活'"——编造的名人名言、用户评价。留 placeholder 问用户要真 quote。

## 六、平面设计专属 Slop

### ❌ 过度使用样机 (Mockup)
- 每个 Logo 都套在名片/杯子/T恤上展示
- 包装效果图用千篇一律的「漂浮立方体」角度
- 品牌规范手册每页都塞满 mockup

**✅ 策略**：Mockup 是辅助，不是主角。1-2 个精妙的应用场景比 10 个 generic mockup 更有说服力。

### ❌ 素材网下载的「通用图标」直接堆砌
- 每段文字前都放一个 flat icon
- 用不同风格的图标混搭（line + filled + colored）
- 图标大小不统一、粗细不一致

**✅ 策略**：统一一套图标风格（全部 line 或全部 filled），统一粗细（1.5px 或 2px），统一尺寸（24×24 或 32×32）。更好的方案：不用图标，用排版和留白解决问题。

### ❌ AI 生成的「手绘风格」插画
AI 生成的「手绘」永远有诡异的比例和重复的细节。要么用真实插画师作品，要么用几何抽象图形替代。

## 七、决策速查：当你犹豫时

| 想做什么 | 决策 |
|---------|------|
| 想加个渐变？ | → 大概率不加。非要加：subtle、单色系、局部点缀 |
| 想加个 emoji？ | → 不加 |
| 想给卡片加圆角+border-left accent？ | → 不加，换背景色对比或留白分隔 |
| 想用 AI 生成人物肖像？ | → 避开面部，用背影/剪影/抽象化 |
| 想加一排 icon features？ | → 先问要不要 icon，可能不需要 |
| 想用 Inter/微软雅黑作标题？ | → 换一个更有特点的 display 字体 |
| 想用紫色渐变？ | → 换一个有根据的配色（品牌色/参考图吸色） |
| 想在海报上塞更多信息？ | → 先做最简版本，用户要求再加 |
| 想放编造的数据/quote？ | → 留 placeholder，问用户要真的 |
| 想把所有内容都居中对称？ | → 试试不对称平衡 |

> **当你觉得「加一下会更好看」的时候——那通常是 AI slop 的征兆**。先做最简的版本，只在用户要求时加。

## 八、正向做什么（带"为什么"）

- ✅ **去色测试**：生成后先转成黑白，检查层级是否仍清晰。如果黑白版乱了，彩色版也一定有问题
- ✅ **oklch() 定义色彩**：比 HSL 更现代，调整亮度时色相不漂移
- ✅ **一个细节做到 120%，其他做到 80%**：品位 = 在合适的地方足够精致，不是均匀用力
- ✅ **3 秒测试**：眯起眼看设计，第一眼看到什么？第二眼？第三眼？如果三眼之内找不到清晰的视觉动线，重新调整层级
- ✅ **文案用「」引号不用 ""**：中文排印规范，也是"有审校过"的细节信号
- ✅ **留白是主动选择，不是没填满**：空白区域需要被设计，有明确的边界和比例
