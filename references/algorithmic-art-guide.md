> **当需要生成品牌辅助图形、包装纹理、动态 Logo 背景、数字艺术元素时，以本文档作为算法美学参照。**

---

## 核心概念

**算法美学 = 计算过程 + 涌现行为 + 数学之美**

不是「随机画一些线条」，而是**用受控的数学规则创造有机的视觉复杂性**。

### 适用场景

| 场景 | 算法方法 | 输出 |
|------|---------|------|
| 品牌辅助图形（底纹/边框）| 噪声场 + 粒子轨迹 | SVG/PNG 图案 |
| 包装纹理（有机/自然感）| 递归细分 + 噪声扰动 | 高分辨率 PNG |
| 动态 Logo 背景 | 粒子系统 + 向量场 | 动画 GIF/MP4 |
| 数据可视化底纹 | Voronoi 镶嵌 + 色彩映射 | 信息图背景 |
| 展览/活动主视觉 | 流场 + 累积轨迹 | 大幅面海报 |
| 社交媒体动态图 | 参数化变化 + 循环 | 无缝循环动画 |

## 一、算法哲学创建框架

在写代码之前，先创建一个算法哲学（类似 canvas-design 的设计哲学，但用计算语言）。

### 命名运动（1-2 个词）

"Organic Turbulence" / "Quantum Harmonics" / "Emergent Stillness" / "Recursive Whispers" / "Field Dynamics" / "Stochastic Crystallization"

### 阐述哲学（4-6 段，每段聚焦一个维度）

| 维度 | 问题 |
|------|------|
| **计算过程** | 什么数学关系驱动视觉？ |
| **噪声与随机** | 使用什么噪声函数？随机性如何被约束？ |
| **粒子行为** | 粒子如何诞生、运动、死亡？ |
| **场动力学** | 什么力场影响粒子/形状？ |
| **时间演化** | 系统如何随时间变化？是静态还是动态？ |
| **参数变化** | 哪些参数可调？变化范围是什么？ |

### 示例："Organic Turbulence"

> 混沌被自然法则约束，无序中涌现秩序。
> 
> 流场由多层 Perlin 噪声驱动。数千粒子跟随向量力，它们的轨迹累积成有机密度图。多层噪声 octave 创造湍流区域和平静区域。颜色从速度和密度中涌现——快速粒子燃烧明亮，慢速的褪入阴影。
> 
> 算法运行直至平衡——一个精心调校的平衡，每个参数都经过无数迭代 refined。

## 二、核心技术工具箱

### 2.1 噪声函数（Noise Functions）

**Perlin Noise** — 最常用，创造有机、连续的随机性
- 适用：地形、云层、水流、有机纹理
- 参数：octaves（细节层数）、persistence（振幅衰减）、lacunarity（频率倍增）

**Simplex Noise** — Perlin 的改进版，计算更快
- 适用：实时生成、大规模粒子系统

**Value Noise** — 简单快速，但连续性较差
- 适用：快速原型、低精度纹理

**Worley Noise** — 细胞状图案
- 适用：Voronoi 纹理、细胞结构、裂纹效果

### 2.2 流场（Flow Fields）

用噪声函数构建 2D 向量场，粒子沿场线运动。

```
for each particle:
  angle = noise(particle.x * scale, particle.y * scale) * TWO_PI
  particle.vx = cos(angle) * speed
  particle.vy = sin(angle) * speed
  particle.move()
  draw trail
```

**变体**：
- **多层流场**：不同 scale 的噪声叠加，创造复杂轨迹
- **旋转流场**：向量场加入旋转向量，创造漩涡
- **吸引/排斥场**：在特定位置添加吸引或排斥力

### 2.3 粒子系统（Particle Systems）

| 属性 | 说明 |
|------|------|
| 诞生 | 随机位置 / 边缘 / 特定区域 |
| 运动 | 沿流场 / 受重力 / 受弹簧力 |
| 寿命 | 固定帧数 / 到达边界死亡 / 速度低于阈值死亡 |
| 视觉 | 点 / 短线 / 轨迹累积 / 发光效果 |

### 2.4 递归结构（Recursive Structures）

**L-Systems** — 用字符串重写规则生成植物/分支结构
- 适用：植物纹理、分支图案、有机生长感

**递归细分** — 不断细分区域，每次加入随机变化
- 适用：镶嵌图案、裂纹、细胞结构

**圆堆积**（Circle Packing）— 随机放置圆，让它们互相推开直至平衡
- 适用：泡沫纹理、气泡背景、有机排列

### 2.5 镶嵌与分割（Tessellation）

**Voronoi 图** — 从随机点生成细胞状分割
- 适用：细胞结构、裂纹、地形图、抽象背景

**Delaunay 三角化** — Voronoi 的对偶图
- 适用：低多边形风格、地形网格、抽象几何

**递归矩形分割** — 不断将矩形分成两个子矩形
- 适用： Mondrian 风格、网格纹理、版式灵感

## 三、色彩策略

### 从算法中涌现色彩

不要随机选 RGB。让颜色从算法的属性中自然产生：

| 算法属性 | 色彩映射 |
|---------|---------|
| 粒子速度 | 快 = 暖色/亮色，慢 = 冷色/暗色 |
| 粒子密度 | 密集 = 高饱和，稀疏 = 低饱和 |
| 距离中心 | 近 = 一种色相，远 = 另一种色相 |
| 噪声值 | 低 = 冷色，高 = 暖色 |
| 递归深度 | 浅 = 亮色，深 = 暗色 |
| 细胞大小 | 大 = 一种色，小 = 另一种色 |

### 调色板生成

1. **固定调色板**：从品牌色中提取 3-5 色，算法只使用这些色
2. **oklch 插值**：用 oklch 色彩空间在两种品牌色之间平滑插值
3. **HSL 变体**：固定色相，变化明度/饱和度

## 四、种子随机性（Seeded Randomness）

> **Art Blocks 模式**：使用种子确保可复现性。

```javascript
let seed = 12345; // 或从用户输入 hash
randomSeed(seed);
noiseSeed(seed);
```

**好处**：
- 同一种子 ALWAYS 产生相同输出
- 用户可以探索种子空间（seed 1-100）找到最喜欢的变体
- 品牌可以固定一个 "官方 seed"

## 五、参数设计

### 好的参数结构

不是「图案类型 A/B/C」，而是**系统属性的连续调节**：

```javascript
let params = {
  seed: 12345,          // 始终包含种子
  // 数量
  particleCount: 5000,  // 多少粒子？
  // 尺度
  noiseScale: 0.01,     // 噪声有多密？
  lineLength: 50,       // 轨迹多长？
  // 概率
  branchingProb: 0.3,   // 分支概率？
  // 比例
  aspectRatio: 1.618,   // 黄金比例？
  // 角度
  baseAngle: 45,        // 基础角度？
  // 阈值
  brightnessThreshold: 0.5, // 何时改变行为？
};
```

### 参数探索建议

为每个品牌项目提供 3-5 个参数变体：
- ** calm（平静）**：低噪声、慢速、少粒子
- **balanced（平衡）**：中等一切
- **chaotic（混沌）**：高噪声、快速、多粒子
- **minimal（极简）**：极少元素、大量留白
- **dense（密集）**：大量元素、高信息密度

## 六、 craftsmanship 标准

> **不是随机噪声——是受控混沌，通过深度专业知识 refined。**

| 维度 | 标准 |
|------|------|
| **Balance** | 复杂但不视觉嘈杂，有序但不僵硬 |
| **Color Harmony** | 深思熟虑的调色板，不是随机 RGB |
| **Composition** | 即使在随机性中，也保持视觉层级和流动 |
| **Performance** | 平滑执行，实时优化 |
| **Reproducibility** | 同一种子始终产生相同输出 |

## 七、执行流程

### 辅助图形生成流程

```
1. 理解品牌需求
   → 这个辅助图形要传达什么？（生长/流动/结构/混沌/秩序？）

2. 创建算法哲学（4-6 段）
   → 命名运动
   → 阐述计算过程、噪声模式、粒子行为、场动力学

3. 选择技术工具
   → 流场？粒子系统？递归？镶嵌？

4. 设定参数
   → 哪些属性可调？范围是多少？

5. 生成并探索
   → 测试多个种子（1-20）
   → 微调参数直至满意

6. 导出
   → 静态：高分辨率 PNG/SVG
   → 动态：GIF/MP4/交互 HTML
```
