## 1. 辅助图形设计指南

> 辅助图形是VI基础系统的重要组成部分，是连接Logo与各类应用物料的"视觉纽带"。

### 1.1 什么是辅助图形

辅助图形是从Logo核心元素中提取、衍生出来的装饰性图形系统，用于：
- 填充空白区域，增强品牌识别度
- 为不同物料提供统一的视觉装饰语言
- 在不直接使用Logo的场合保持品牌调性

### 1.2 辅助图形设计方法

| 方法 | 说明 | 适用场景 | 示例 |
|------|------|---------|------|
| **元素提取法** | 从Logo中提取1-2个核心形状/线条，进行旋转/重复/缩放 | 大多数品牌 | 从圆形Logo中提取圆弧，做波浪底纹 |
| **网格衍生法** | 基于Logo的网格系统生成辅助几何图案 | 科技/金融品牌 | IBM的横线条纹系统 |
| **意象延伸法** | 从品牌名称/行业意象中提取视觉元素 | 文化/餐饮/自然品牌 | 咖啡品牌提取蒸汽曲线 |
| **动态变体法** | 设计一套参数化图形，不同物料使用不同参数版本 | 科技/互联网品牌 | Google的动态Logo变体 |

### 1.3 辅助图形设计原则

| 原则 | 说明 | 常见错误 |
|------|------|---------|
| **简约性** | 1-2套简洁样式，不超过3种变化维度 | 设计过于复杂的图案系统 |
| **灵活性** | 可横向/纵向延伸，适配不同尺寸的物料 | 只能在特定比例下使用 |
| **克制性** | 辅助图形≤画面面积的20-30%，不抢Logo风头 | 辅助图形过于醒目，喧宾夺主 |
| **一致性** | 所有辅助图形共享同一视觉基因 | 不同物料使用风格不统一的辅助图形 |
| **可缩放** | 大到灯箱、小到名片都能使用 | 细节过多，缩小后模糊 |

### 1.4 辅助图形分类设计

| 类型 | 用途 | 设计要点 |
|------|------|---------|
| **线条类** | 边框、分割线、装饰线 | 1-3种线宽，与Logo线条粗细协调 |
| **底纹类** | 背景、包装底纹、信纸底纹 | 透明度5-15%，低对比度，不干扰文字阅读 |
| **图形类** | 重复排列的几何形状 | 保持规律性，间距统一 |
| **组合类** | 图形+文字的组合装饰 | 文字使用品牌字体，图形使用辅助图形元素 |

### 1.5 辅助图形使用禁忌

- ❌ 不得修改辅助图形的比例关系
- ❌ 不得使用与品牌色系不符的颜色
- ❌ 不得将辅助图形作为独立Logo使用
- ❌ 辅助图形不得遮挡Logo的安全空间
- ❌ 不得在辅助图形上叠加重要文字信息

## 2. 字体选择决策树

> 为非专业设计师提供清晰的字体选择路径，确保选对字体、避免版权风险。

### 2.1 字体选择决策流程

```
第一步：确定用途
├── Logo → 选装饰性/特征性字体（如站酷小薇LOGO体）→ 转为矢量路径
├── 标题 → 选中等/粗字重无衬线体（如思源黑体 Bold）
├── 正文 → 选常规字重、高可读性字体（如思源黑体 Regular）
└── 注释 → 选细字重、小字号友好字体（如思源黑体 Light）

第二步：确定行业调性
├── 科技/互联网 → 几何无衬线（思源黑体/OPPO Sans）
├── 金融/法律 → 经典稳重（思源宋体/方正黑体）
├── 餐饮/零售 → 圆润友好（阿里巴巴普惠体/江城正义体）
├── 文化/教育 → 人文气质（思源宋体/站酷快乐体）
├── 奢侈/高端 → 纤细优雅（思源宋体 Light + 衬线英文）
└── 儿童/活泼 → 趣味手写（站酷快乐体/站酷小薇LOGO体）

第三步：中西文匹配
├── 思源黑体 ← → Inter / Helvetica Now / Roboto
├── 思源宋体 ← → Merriweather / Lora / Playfair Display
├── 阿里巴巴普惠体 ← → Open Sans / Nunito
└── 方正黑体 ← → Arial / Univers

第四步：版权检查
├── 在安全字体清单中？ → ✅ 可用
├── 客户提供授权？ → ✅ 可用
└── 都不是？ → ❌ 换用安全字体或购买授权
```

### 2.2 字体搭配速查

| 搭配方式 | 中文标题 | 中文正文 | 英文标题 | 英文正文 | 适用风格 |
|----------|---------|---------|---------|---------|---------|
| **经典安全** | 思源黑体 Bold | 思源黑体 Regular | Inter Bold | Inter Regular | 通用/商务 |
| **文化质感** | 思源宋体 Bold | 思源宋体 Regular | Merriweather Bold | Merriweather Regular | 文化/出版 |
| **现代科技** | OPPO Sans Bold | OPPO Sans Regular | Roboto Bold | Roboto Regular | 科技/互联网 |
| **活泼亲和** | 站酷快乐体 | 思源黑体 Regular | Nunito Bold | Nunito Regular | 儿童/餐饮 |
| **极简高端** | 思源黑体 Medium | 思源黑体 Light | Helvetica Now Medium | Helvetica Now Light | 奢侈/设计 |

### 2.3 字体使用刚性规范

| 规范 | 要求 | 原因 |
|------|------|------|
| **字体种类** | 同一物料 ≤ 2种字体家族 | 超过3种字体=视觉混乱 |
| **字重变化** | 同一家族用不同字重区分层级 | 比换字体更和谐 |
| **最小字号** | 印刷正文 ≥ 8pt，数字端 ≥ 14px | 低于此影响可读性 |
| **行距** | 正文行距 1.3-1.6倍字号 | 太紧=压抑，太松=散乱 |
| **行长** | 中文正文 ≤ 35字/行，英文 ≤ 75字符/行 | 过长导致阅读困难 |
| **禁用操作** | 不得随意加粗、改字间距、拉伸文字 | 破坏字体设计美感 |

## 3. 竞品视觉分析模板

> 品牌VI设计前必须做竞品视觉分析，确保差异化定位。

### 3.1 竞品分析框架

| 分析维度 | 检查项 | 评分（1-5） |
|----------|--------|------------|
| **Logo风格** | 字标/图标/组合？抽象/具象？几何/有机？ |  |
| **主色系** | 色相/饱和度/明度？冷色/暖色？单色/多色？ |  |
| **字体选择** | 衬线/无衬线？粗/细？中文/英文风格？ |  |
| **视觉风格** | 极简/装饰/科技/自然/奢华/活泼？ |  |
| **辅助图形** | 有无？什么类型？使用频率？ |  |
| **物料统一性** | 不同物料风格是否一致？ |  |
| **整体印象** | 专业感/亲和力/科技感/文化感？ |  |

## 4. Moodboard制作指南

> Moodboard（情绪板）是品牌设计前的视觉参考收集工具，帮助在设计开始前就明确方向。

### 4.1 什么是Moodboard

Moodboard是一组图片、色彩、字体、纹理、构图的集合，用来传达设计的"感觉方向"，不涉及具体设计。

### 4.2 Moodboard收集维度

| 维度 | 收集内容 | 数量 |
|------|---------|------|
| **色彩氛围** | 色彩组合、渐变、色调参考图 | 3-5张 |
| **字体气质** | 字体排版参考、文字组合方式 | 2-3张 |
| **构图方式** | 版式、留白、对齐参考 | 3-5张 |
| **纹理材质** | 背景、纸张、金属、布料等质感 | 2-3张 |
| **摄影风格** | 图片色调、构图、氛围 | 3-5张 |
| **整体调性** | 能代表品牌"感觉"的参考图 | 2-3张 |

### 4.3 Moodboard制作步骤

1. **确定关键词**：从品牌定位推导3-5个视觉关键词（如"温暖、手工、自然"）
2. **收集素材**：按上述维度搜索和收集参考图片
3. **筛选精简**：从收集的素材中选出15-20张最能代表方向的
4. **排列组合**：将素材排列成网格/自由式布局
5. **提取色板**：从Moodboard中提取5-8种主色，形成初步色板
6. **验证方向**：与用户确认Moodboard传达的感觉是否正确

### 4.4 AI辅助Moodboard制作

在 my-desgin 工作流中，Moodboard可通过以下方式实现：
- **搜索参考图**：使用 `web_search` 搜索行业视觉参考
- **色彩提取**：使用 `color-palette` 从参考图提取色彩
- **情绪板生成**：使用 Seedream 5.0 生成 moodboard 风格的参考图
- **布局展示**：使用 canvas-design 制作 Moodboard 展示板

## 5. 色彩使用比例规范

> 品牌色彩不是"有什么用什么"，而是有严格的使用比例规范。

### 5.1 色彩使用60-30-10法则

| 角色 | 比例 | 说明 | 典型用途 |
|------|------|------|---------|
| **主色** | 60% | 占据视觉面积最大 | 背景、大面积色块、主视觉 |
| **辅助色** | 30% | 补充层次和节奏 | 标题、图形、次要色块 |
| **强调色** | 10% | 吸引注意力的焦点 | CTA按钮、重点信息、装饰 |

### 5.2 各场景色彩比例适配

| 场景 | 主色占比 | 辅助色占比 | 强调色占比 | 说明 |
|------|---------|-----------|-----------|------|
| 海报 | 50-60% | 25-35% | 10-15% | 可适当提高强调色比例 |
| 名片 | 40-50% | 30-40% | 10-20% | 小面积需要更多强调 |
| 网站 | 60-70% | 20-30% | 5-10% | 大面积使用主色保持统一 |
| 招牌 | 55-65% | 25-35% | 10-15% | 远距离识别需高对比 |
| 包装 | 50-60% | 25-35% | 10-20% | 货架展示需突出 |

### 5.3 色彩使用禁忌

- ❌ 不得将辅助色作为大面积背景色使用
- ❌ 同一画面不得出现3种以上高饱和度色彩
- ❌ 强调色不得用于大面积区域（失去强调作用）
- ❌ 不得在品牌主色上叠加透明度变化创造"新色"
- ❌ 中性色（黑白灰）不计入色彩比例，但过度使用会导致品牌感缺失

## 6. VI规范维护指南

> VI不是做完就结束的，需要长期维护和迭代。

### 6.1 VI维护三阶段

| 阶段 | 时间 | 核心任务 | 责任方 |
|------|------|---------|--------|
| **交付期** | 项目结束后1个月 | 培训相关人员、答疑、修正交付物中的问题 | 设计方 |
| **执行期** | 1-12个月 | 监督物料制作是否遵循规范、定期抽查 | 品牌方+设计方 |
| **迭代期** | 1年以上 | 根据品牌发展需要小调整、补充新场景规范 | 品牌方 |

### 6.2 VI规范更新触发条件

| 触发条件 | 调整级别 | 说明 |
|----------|---------|------|
| 新增应用场景（如新开社交媒体账号） | 🟢 小调整 | 补充新场景规范 |
| 现有规范无法覆盖新需求 | 🟡 中调整 | 修订规范文档 |
| 品牌战略调整 | 🔴 大调整 | 重新审视视觉系统 |
| Logo/主色需变更 | 🔴 重大更新 | 等同于重新做VI |

### 6.3 VI规范文档版本管理

- 每次更新记录版本号（如 v1.0 → v1.1 → v2.0）
- 小调整递增0.1，重大更新递增1.0
- 每个版本保留变更日志
- 过期版本归档但不删除

## 7. 落地执行与打样检查清单

> 设计稿到实物落地是最容易出问题的环节，此清单帮助把关。

### 7.1 印刷前检查清单

| 检查项 | 要求 | 常见问题 |
|--------|------|---------|
| 色彩模式 | 印刷品使用CMYK模式 | 误用RGB模式导致色差 |
| 分辨率 | 位图≥300dpi | 低分辨率导致模糊 |
| 出血线 | 四周各加3mm出血 | 忘记出血导致裁切露白边 |
| 文字转曲 | 印刷文件文字转曲/嵌入 | 字体缺失导致文字变方块 |
| 专色设定 | 需要专色时正确设置Pantone | 专色与四色混用导致色差 |
| 线条粗细 | 细线≥0.25pt | 细线太细印刷不出 |

### 7.2 打样确认要点

| 确认项 | 检查方法 | 通过标准 |
|--------|---------|---------|
| **色彩准确性** | 对照色卡/Pantone对比 | CMYK偏差≤5%，专色偏差≤3% |
| **文字清晰度** | 肉眼检查最小字号 | 8pt以上文字清晰可读 |
| **Logo完整性** | 对照VI规范检查 | 尺寸/色彩/安全空间全部合规 |
| **材质匹配** | 触摸感受/对比样品 | 材质与设计意图一致 |
| **工艺效果** | 检查烫金/UV/凹凸等特殊工艺 | 工艺位置准确、效果达标 |

### 7.3 数字物料检查清单

| 检查项 | 要求 | 常见问题 |
|--------|------|---------|
| 色彩模式 | 屏幕展示使用sRGB模式 | 使用CMYK导致屏幕色彩暗淡 |
| 图片尺寸 | 2倍实际显示尺寸（Retina适配） | 1倍尺寸在高分屏模糊 |
| 文件格式 | Web用WebP/PNG，社交用JPG/PNG | 格式不当导致质量/体积问题 |
| Favicon | 16×16 + 32×32 + 180×180 | 仅做一个尺寸导致兼容问题 |
| 社交预览图 | 1200×630px（OG标准） | 尺寸不当导致裁切 |
