# 审美学习与风格固化机制（Aesthetic Learning & Style Fixation）

> 硬规则文件。与 SKILL.md 同级约束力。违反 = 风格漂移。
>
> **核心目的**：在出品过程中学习设计师（用户）的审美偏好，将偏好从"临时意见"固化为"可执行规则"，让后续产出自动对齐用户审美，而不是每次都靠用户纠正。

---

## 1. 核心问题

VI 设计过程中，设计师会不断表达审美偏好：

- "绿色太暗了" → 偏好中高明度
- "这个太花了" → 偏好克制留白
- "不够精致" → 偏好细腻工艺感
- "感觉不对" → 偏好某种说不清的氛围

**如果这些偏好只在当前对话中起作用**，就会出现：
- 下个物料又犯同样的审美错误
- 用户每次都要重新解释"我想要什么样的感觉"
- AI 的审美选择和用户的审美偏好持续摩擦
- 最终产出风格不统一——有的物料对味，有的不对味

**本机制要解决的核心问题：把用户的审美偏好从"每次都要说"变成"说了就固化，以后自动遵守"。**

---

## 2. 硬规则体系

### R-A1: 审美偏好即时捕获

**规则**：用户在出品过程中表达的任何审美偏好，无论显式还是隐式，**必须立即捕获并分类记录**。

**捕获信号识别**：

| 用户表述类型 | 示例 | 偏好归类 |
|-------------|------|---------|
| 直接评价 | "这个好看"/"这个不好看" | 整体偏好 |
| 具体批评 | "颜色太暗了"/"太花了"/"太素了" | 特定维度偏好 |
| 比较偏好 | "B方案比A方案好看" | 相对偏好 |
| 感觉描述 | "不够高级"/"不够有活力"/"太严肃了" | 氛围偏好 |
| 参考指向 | "我觉得Shake Shack那种感觉" | 参考偏好 |
| 否定偏好 | "不要那种很潮的感觉"/"不要渐变" | 禁止偏好 |
| 反复选择 | 多次选择同一类型方案 | 隐式偏好（需推导） |

**捕获后立即分类**：

| 维度 | 包含内容 | 记录位置 |
|------|---------|---------|
| 色彩偏好 | 明度/饱和度/色相倾向、冷暖倾向、特定颜色偏好/禁止 | brand-profile.md 审美档案 |
| 排版偏好 | 留白量、对齐方式、密度偏好 | brand-profile.md 审美档案 |
| 字体偏好 | 衬线/无衬线、字重、中文/英文匹配 | brand-profile.md 审美档案 |
| 构图偏好 | 对称/不对称、中心/偏移、层次数量 | brand-profile.md 审美档案 |
| 风格偏好 | 极简/装饰、现代/经典、科技/自然 | brand-profile.md 审美档案 |
| 氛围偏好 | 高级/亲民、活力/沉静、温暖/冷静 | brand-profile.md 审美档案 |
| 禁止偏好 | 明确不想要的元素/风格/色彩 | brand-profile.md 禁止清单 |

### R-A2: 偏好翻译（模糊感受→可执行参数）

**规则**：用户的审美偏好往往是模糊的（"再高级一点"/"不够有活力"），必须翻译为**具体可执行的视觉参数**。

**翻译对照表**：

| 用户模糊表述 | → 翻译为视觉参数 | 参数示例 |
|-------------|-----------------|---------|
| "太暗了" | 提升明度 | 明度+15%，HSL的L值上调 |
| "太亮了" | 降低明度 | 明度-10% |
| "太花了" | 减少色彩数量/降低饱和度 | 主色≤2种，饱和度降至70%以下 |
| "太素了" | 增加层次/添加辅助图形 | 增加辅助图形占比至15% |
| "不够高级" | 提升留白、降低饱和度、换衬线体 | 留白≥40%，饱和度≤60% |
| "不够有活力" | 提升饱和度/明度、加活力色 | 主色饱和度≥70%，加活力强调色 |
| "太严肃了" | 增加圆角/曲线/暖色 | 圆角半径+50%，暖色占比提升 |
| "不够精致" | 减少粗线条、增加细节、提升对比 | 线宽≤1pt，添加精细装饰 |
| "太粗糙" | 提升工艺描述词 | prompt追加meticulously crafted, master-level |
| "不要AI味" | 减少渐变/发光/金属质感 | 禁用渐变，平面矢量风格 |

**翻译后必须向用户确认**：

```
🎨 审美偏好翻译
━━━━━━━━━━━━━━━━━━━
你说："[用户原文]"
我理解为：
  - [具体参数变化1]
  - [具体参数变化2]
  - [具体参数变化3]
对吗？还是我理解偏了？
```

### R-A3: 偏好固化到品牌审美档案

**规则**：确认后的审美偏好，必须**立即写入** `brand-profile.md` 的「品牌审美档案」区域，不是只在对话中记住。

**品牌审美档案结构**：

```markdown
## 🎨 品牌审美档案

### 色彩偏好
| 偏好 | 参数 | 来源 |
|------|------|------|
| 中高明度绿色系 | 主色明度≥45% | AP-001 |
| 避免低饱和冷色 | 饱和度≥50%，色相≤180° | AP-002 |

### 排版偏好
| 偏好 | 参数 | 来源 |
|------|------|------|
| 大留白 | 留白≥35% | AP-003 |

### 风格偏好
| 偏好 | 参数 | 来源 |
|------|------|------|
| 现代+有机感结合 | 几何线条+自然曲线混用 | AP-004 |

### 氛围偏好
| 偏好 | 参数 | 来源 |
|------|------|------|
| 活力但不廉价 | 鲜明色彩+克制留白 | AP-005 |

### 禁止偏好
| 禁止项 | 原因 | 来源 |
|--------|------|------|
| 低明度冷绿色 | 用户评价"像医院不像餐厅" | AP-001 |
| 渐变发光效果 | 用户评价"有AI味" | AP-006 |
```

### R-A4: 审美档案驱动后续产出

**规则**：品牌审美档案中的偏好，与 brand-profile.md 的锁定项**同级约束力**。后续所有产出的 prompt、SVG、画布设计，必须主动应用审美档案中的偏好。

**执行方式**：

1. **每个物料生成前**，读取 brand-profile.md 的审美档案
2. **构造 prompt 时**，将偏好翻译为 prompt 关键词：
   - 色彩偏好 → prompt 中的色彩描述
   - 风格偏好 → prompt 中的风格关键词
   - 禁止偏好 → prompt 中的 negative prompt
3. **SVG 绘制时**，将偏好翻译为 SVG 参数
4. **画布排版时**，将偏好翻译为排版参数

**一致性自检增加审美维度**：

```
✅ 品牌名是否与 brand-profile.md 逐字一致？
✅ 色彩是否使用 brand-profile.md 中的 HEX 值？
✅ 审美偏好是否已应用到本物料？（新增）
✅ 禁止偏好是否已规避？（新增）
✅ 氛围感是否与审美档案一致？（新增）
```

### R-A5: 偏好累积与冲突检测

**规则**：随着出品过程推进，审美档案会累积越来越多偏好。必须检测偏好之间是否有冲突。

**冲突检测场景**：

| 冲突类型 | 示例 | 处理方式 |
|----------|------|---------|
| 新偏好 vs 旧偏好 | 之前偏好留白≥35%，现在说"太素了" | 询问用户：调整参数还是推翻旧偏好？ |
| 色彩偏好 vs 色彩锁定 | 偏好暖色但锁定色是绿色 | 绿色是锁定项，在绿色范围内调整明度/饱和度 |
| 风格偏好 vs 设计哲学 | 偏好极简但设计哲学是"活力脉搏" | 在设计哲学框架内实现极简+活力 |

**冲突处理流程**：
1. 检测到冲突
2. 输出冲突声明（两个偏好哪个优先？）
3. 等用户确认
4. 按确认结果更新审美档案

### R-A6: 审美纠偏闭环

**规则**：如果用户纠正了 AI 的审美选择，必须执行闭环：纠正→翻译→固化→更新禁止清单→验证。

**闭环流程**：

```
用户纠正审美选择
      ↓
  ┌──────────────────────┐
  │ R-A1: 捕获纠正原文     │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ R-A2: 翻译为可执行参数  │
  │ 向用户确认翻译是否准确   │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ R-A3: 写入审美档案      │
  │ 记录偏好+参数+来源编号   │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ 更新禁止清单            │
  │ 将"不对的"加入禁止      │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ R-A4: 验证后续产出对齐   │
  │ 下一个物料必须体现新偏好  │
  └──────────────────────┘
```

### R-A7: 审美档案编号追溯

**规则**：审美档案中的每条偏好必须有编号（AP-001, AP-002, ...），可追溯到用户的原始表述。

**目的**：
- 当偏好之间冲突时，可追溯哪个偏好先确认的
- 当偏好需要推翻时，可精确定位要推翻的条目
- 当新 session 开始时，可快速理解所有偏好的来龙去脉

## 2.5 参考作品投喂与审美学习（R-A8 ~ R-A10）

> **用户可以投喂参考作品（网上找的、自己过去的、后续满意的），skill 必须从中主动提取审美特征并固化到审美档案。**

### R-A8: 参考作品接收与分类

**规则**：当用户投喂图片作品时，必须先对作品分类，再提取审美特征。

**投喂来源分类**：

| 来源类型 | 说明 | 学习权重 | 编号前缀 |
|----------|------|---------|---------|
| 🌐 网上参考 | 用户从网上找到的喜欢的设计作品 | 高（用户主动选择=强烈偏好信号） | REF-WEB |
| 🎨 历史作品 | 用户自己过去的满意作品 | 最高（自己的作品=审美DNA） | REF-OWN |
| ✅ 项目产出 | 本次项目中用户满意的产出 | 高（已验证的审美共识） | REF-PRO |
| 🚫 反面参考 | 用户明确说"不要这种"的作品 | 特殊（提取禁止偏好） | REF-NEG |

**接收流程**：

```
用户投喂图片（单张或多张）
      ↓
  ┌──────────────────────┐
  │ 询问：这张作品的来源？   │
  │ A. 网上找的参考         │
  │ B. 我自己的作品         │
  │ C. 你刚做的，我很满意    │
  │ D. 反面教材，不要这种     │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ 询问：你喜欢这张的什么？  │
  │ （多选）               │
  │ □ 整体氛围/感觉         │
  │ □ 色彩搭配              │
  │ □ 排版布局              │
  │ □ 字体选择              │
  │ □ 图形/插画风格         │
  │ □ 构图方式              │
  │ □ 材质/纹理             │
  │ □ 全都喜欢/说不清       │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ R-A9: 提取审美特征      │
  └──────────────────────┘
```

**快捷投喂**：如果用户一次性投喂多张图片，可以先批量接收，逐张提取，最后汇总分析。

### R-A9: 参考作品审美特征提取

**规则**：收到参考作品后，必须使用 color-palette + 视觉分析提取以下审美特征，并翻译为可执行参数。

**提取维度与方法**：

| 维度 | 提取方法 | 输出参数 |
|------|---------|---------|
| **色彩** | 使用 color-palette 技能提取主色调 → 分析色相/明度/饱和度分布 | 偏好色相范围、明度范围、饱和度范围、冷暖倾向、配色模式（互补/类似/三角等） |
| **排版** | 观察留白比例、对齐方式、元素密度 | 留白比例偏好、对齐偏好、密度偏好 |
| **字体** | 识别字体类型（衬线/无衬线/手写/装饰）、字重、字间距 | 字体类型偏好、字重偏好、中文西文匹配方式 |
| **图形风格** | 判断几何/有机/手绘/照片、扁平/立体、抽象/具象 | 图形风格偏好、扁平度偏好 |
| **构图** | 分析对称性、视觉重心、层次数量 | 构图偏好、层次偏好 |
| **氛围** | 综合判断：高级/亲民、活力/沉静、温暖/冷静、现代/经典 | 氛围关键词 |
| **纹理/材质** | 识别：纯色/渐变/噪点/纸张/金属/自然 | 材质偏好 |

**color-palette 集成**：

使用 `~/.workbuddy/skills/color-palette/` 提取参考作品的色彩：

```bash
# 从参考图提取色彩
python ~/.workbuddy/skills/color-palette/scripts/extract_palette.py [图片路径] --num-colors 8 --output-format hex
```

**特征提取输出格式**：

```
🔍 参考作品审美特征提取
━━━━━━━━━━━━━━━━━━━━━━━
作品编号：REF-WEB-001
来源：网上参考

| 维度 | 特征 | 可执行参数 |
|------|------|-----------|
| 色彩 | 主色调偏中高明度绿色系，辅色为暖白，强调色为金色 | 主色明度45-55%、饱和度60-75%、色相100-160° |
| 排版 | 大留白，居中对齐，元素稀疏 | 留白≥40%、居中构图 |
| 字体 | 粗无衬线西文 + 中等粗细中文无衬线 | 西文Bold+中文Medium |
| 图形 | 扁平矢量，圆角，简洁 | 圆角半径≥8px、扁平风格 |
| 构图 | 单一视觉焦点，2层视觉层级 | 1个焦点+2层层级 |
| 氛围 | 现代、健康、高级 | 关键词：modern, organic, premium |
| 材质 | 纯色为主，无渐变 | 禁用渐变 |

与当前品牌审美档案的一致性：[高/中/低]
冲突项：[如有，列出]
```

### R-A10: 参考作品审美特征固化

**规则**：从参考作品中提取的审美特征，必须按以下流程固化到品牌审美档案。

**固化流程**：

```
R-A9 提取审美特征
      ↓
  ┌──────────────────────┐
  │ 与现有审美档案对比       │
  │ - 一致的→强化权重        │
  │ - 新增的→追加偏好        │
  │ - 冲突的→R-A5冲突处理    │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ 向用户确认提取结果       │
  │ "我从这张图中读到：     │
  │  - 中高明度绿色系偏好    │
  │  - 大留白排版偏好       │
  │  - 扁平圆角图形偏好     │
  │  对吗？"               │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ 写入 brand-profile.md  │
  │ 审美档案区域            │
  │ 追加 AP 编号条目        │
  │ 来源标注为 REF-XXX     │
  └───────┬──────────────┘
          ↓
  ┌──────────────────────┐
  │ 保存参考图到项目目录     │
  │ /references/           │
  │ 命名：REF-XXX-描述.png  │
  └──────────────────────┘
```

**参考图归档规则**：

| 规则 | 说明 |
|------|------|
| 保存位置 | 项目目录 `/aesthetic-references/` |
| 命名格式 | `REF-{类型}-{编号}-{简述}.png`，如 `REF-WEB-001-shake-shack-menu.png` |
| 伴生文件 | 同名 `.md` 文件记录提取的审美特征，如 `REF-WEB-001-shake-shack-menu.md` |
| 审美档案引用 | brand-profile.md 中的偏好条目，来源列标注 REF 编号 |

**伴生 .md 文件格式**：

```markdown
# REF-WEB-001: Shake Shack 菜单设计

## 基本信息
| 字段 | 内容 |
|------|------|
| 来源 | 网上参考 |

| 用户关注的维度 | 整体氛围、色彩、排版 |

## 提取的审美特征
| 维度 | 特征 | 可执行参数 |
|------|------|-----------|
| 色彩 | ... | ... |
| 排版 | ... | ... |

## 固化到审美档案的偏好
- AP-005: [偏好内容]（来源：REF-WEB-001）
```

## 3. 与其他机制的关系

| 机制 | 管什么 | 与审美学习的关系 |
|------|--------|----------------|
| **品牌一致性铁律（R1-R7）** | 锁定项不可变 | 审美偏好不能推翻锁定项（如品牌色），但在锁定范围内可以调整 |
| **Skill流程自进化（R-E1~R-E7）** | 规则文件迭代 | 如果审美偏好导致流程调整，触发自进化 |
| **审美学习与风格固化（R-A1~R-A10）** | 审美偏好固化 | 本机制本身 |
| **参考作品投喂（R-A8~R-A10）** | 从作品学习审美 | 审美学习的"主动输入"通道，R-A1~R-A7是"被动捕获"通道 |

**核心关系**：
- 铁律管"绝对不能变什么"
- 自进化管"规则怎么迭代"
- 审美学习管"用户喜欢什么"——两个通道：
  - R-A1~R-A7：从对话中被动捕获偏好
  - R-A8~R-A10：从参考作品中主动学习偏好
- 修改管理管"怎么安全地改"

## 4. 与 brand-profile.md 的集成

brand-profile.md 文件结构应包含以下审美相关区域：

```markdown
## 🎨 品牌审美档案

### 审美偏好记录
[AP编号表]
```
