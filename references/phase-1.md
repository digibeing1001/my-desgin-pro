# Phase 1: 需求追问

> 本文件为 Phase 1 的完整指令。由 SKILL.md 路由按需加载，不默认注入上下文。

收到设计需求后，通过结构化追问理清关键信息，**不要直接生成**。

## 必问项

| 维度 | 追问内容 | 目的 |
|------|---------|------|
| 品牌信息 | 品牌名/公司名？有无现有logo/VI？ | 确定视觉基础 |
| 品牌定位 | 目标受众是谁？核心竞品？品牌个性关键词？ | 确定品牌战略方向（brand-cog） |
| 设计类型 | 海报？名片？招牌？整套VI？ | 确定产出范围 |
| 行业/场景 | 什么行业？用在什么场合？ | 影响风格与合规 |
| 风格偏好 | 极简？国潮？科技感？温馨？ | 引导视觉方向 |
| 色彩倾向 | 有品牌色吗？偏好暖色/冷色？ | 锁定配色 |
| 文字内容 | 需要出现哪些文字？中英文？ | 合规审查前置 |
| 尺寸/比例 | 有明确尺寸要求吗？ | 确定生图尺寸 |
| 输出格式 | SVG矢量？PSD？PNG？ | 确定后处理流程 |
| 字体授权 | 有无已购买授权的字体？需指定字体吗？ | 避免字体版权侵权 |
| 商标注册 | 品牌是否已注册商标？®还是™？ | 商标使用合规 |

## 追问策略

- 一次最多问3-4个问题，避免用户压力
- 根据用户回答动态调整后续问题
- 用户已有明确意向的维度跳过
- 追问结束后，用一段话总结需求确认

## 关键提示

- 品牌名确认后立即创建brand-profile.md，不要等到Phase 2
- 如果用户只要求做单张海报，不需要全套VI追问，聚焦海报相关信息
- 追问中如果用户提到行业关键词（如"有机""医疗""金融"），应标记为合规重点关注项
