# Phase 1: 需求追问

> 本文件为 Phase 1 的完整指令。由 SKILL.md 路由按需加载，不默认注入上下文。
> **执行原则：按 Layer 递进，过 Gate 解锁。禁止一次性抛出全部问题。**

---

## Layer 1 · 表面层（方向确认）

> **目标：用最少的问题，确认设计的大方向。不给用户压力，不追问细节。**

### 此刻只问这 4 个维度（最多 4 个）：

| # | 维度 | 问题 | 目的 |
|---|------|------|------|
| 1 | **品牌身份** | 品牌/公司叫什么？ | 确定品牌名锁定项 |
| 2 | **设计类型** | 要做的是什么？（一张海报 / 一个Logo / 整套VI / 包装…） | 确定模式路由 |
| 3 | **核心目的** | 这个设计要解决什么问题？（吸引顾客？建立信任？活动宣传？） | 区分目标与手段 |
| 4 | **风格倾向** | 有特别喜欢的风格方向吗？（或提供参考图） | 确定审美起点 |

> **新增：资产预检（R15）** — 在方向确认后、进入 Layer 2 前，必须追加一问：
> 
> ```
> 你手上是否有以下资料？有的话直接发我，我会在后续设计中优先使用：
> 1. Logo（JPG/PNG/SVG）—— 如有，我直接基于你的 Logo 做扩展，不重新生成
> 2. 产品图 / 官方渲染图 / 产品草图 / 竞品照片 / 灵感图
> 3. UI 截图 / 界面素材
> 4. 品牌色 / 字体清单 / Brand guidelines
> ```
> 
> **执行规则**：用户上传任何文件 → 立即保存到 `assets/user-uploads/` → 在 `brand-profile.md` 和 `brand-spec.md` 中登记来源为「用户提供」→ 后续所有产出强制引用。
> 
> **参考图特殊处理（产品渲染模式）**：
> - 若用户上传了产品相关参考图（草图/照片/竞品图），**立即向用户描述图中内容并请其确认**
> - 提取关键视觉特征（形状/材质/颜色/风格）作为产品访谈的**默认值**
> - 有参考图已回答的问题，在产品访谈中**跳过**
> - 参考图路径登记到 `product-fingerprint.md` 的「参考图」字段
> - 后续生图优先使用图生图模式（`image` 参数），而非纯文生图 |

### Layer 1 执行规则

- **一次只问 1-2 个问题**，不要一次性列出 4 个
- 用户回答后，**用一句话重述你的理解**，确认方向正确
- 如果用户说"还没想好"或"你看着办"→ 进入 PM-Clarity 的「手段伪装成目标」追问（见 SKILL.md Phase 1 Clarify 追问法）

## Layer 2 · 细节层（信息补全）

> **目标：根据已确认的方向，补全执行所需的精确参数。**
> **仅在用户通过 P1-G1 后解锁。**

### 2.1 设计执行语境预检（融入huashu-design「位置四问」）

执行「位置四问」框架，建立设计执行语境。见 SKILL.md §位置四问。

记录到 brand-profile.md 的「执行语境」字段。

### 此刻追问剩余维度（动态选择，不是全问）：

| 维度 | 追问内容 | 何时必问 |
|------|---------|---------|
| 目标受众 | 给谁看？年龄/职业/消费层级？ | 品牌类必问 |
| 核心竞品 | 有哪些直接竞品？ | 品牌类必问 |
| 行业/场景 | 什么行业？用在什么具体场合？ | 涉及合规必问（医疗/食品/金融等） |
| 色彩倾向 | 有品牌色吗？偏好暖/冷/高饱和/低饱和？ | 用户未提供参考图时必问 |
| 文字内容 | 需要出现哪些文字？中英文比例？ | 所有物料必问 |
| 尺寸/比例 | 有明确尺寸吗？（A4/名片/方形/横幅） | 除Logo外必问 |
| **场景照片/空间尺寸** | **用于前台/门头/展墙等空间设计：请提供现场照片（多角度）。墙面实际宽高是多少？有已知尺寸参照物（如椅子/A4纸/卷尺）吗？** | **空间类物料必问（R17）** |
| **产品类设计** | **产品效果图/电商主图/详情页/包装产品图：进入 SKILL.md §产品类设计追问清单（15问），创建 PIF** | **🟣 产品渲染模式必问** |
| 输出格式 | 需要源文件吗？（SVG/PSD/仅PNG） | 必问 |
| 字体授权 | 有已购字体吗？需指定字体吗？ | 商用必问 |
| 商标注册 | 品牌是否已注册商标？®还是™？ | 品牌类必问 |

### Layer 2 执行规则

- **根据设计类型筛选问题**：单张海报不问「核心竞品」，整套VI必问
- **每轮追问 ≤3 个问题**，分批进行
- 每轮回答后，**更新需求简报**（见 SKILL.md 需求简报模板）
- **精确度评分**：每轮追问后自评（0-3分），<2分继续追问
- 到达追问上限（轻量3轮/中等5轮/深度8轮）时，输出需求简报并问"还差什么没聊到？"
- **顾问模式触发（R22）**：当用户对材质/工艺/品类回答"不知道""你推荐""看情况""随便"时：
  1. **立即停止当前追问**
  2. 读取 `references/material-guide.md` 对应品类
  3. 提供 **2–3 个选项**，格式：材质+工艺 + 适用理由 + 优点 + 局限 + 参考价格
  4. 等用户**明确选择**后方可继续
  5. **禁止不提供选项就直接替用户决定**

```
2. 是否推进到 Phase 2（竞品分析+设计哲学）？（是 / 否）
```

## Layer 3 · 高级层（深度定制）

> **目标：处理边缘情况和深度需求。**
> **仅在用户主动提出或场景触发时解锁。**

### 触发条件

- 用户说"我们还有…"、"另外…"、"能不能也…"
- 涉及多语言版本、多尺寸适配、动态Logo
- 用户提到「品牌升级」「品牌重塑」（需要旧品牌资产盘点）
- 用户提到「系列物料」「批量生成」

### Layer 3 内容

- 旧品牌资产盘点（升级/重塑场景）
- 多语言/多区域适配需求
- 动态/交互式标识需求
- 系列物料的延展策略
- 预算和时间约束下的优先级调整

### P1-G3 · Phase 验收门

Phase 1 最终验收，更新 `brand-profile.md` 的 Layer 1 字段为最终值：

```
📍 Phase 1 完成 · 最终确认
━━━━━━━━━━━━━━━━━━━
品牌档案已创建：{brand-profile.md 路径}
已锁定项：品牌中英文名称

请确认：
1. brand-profile.md 中的品牌名是否正确？（是 / 否，需修改）
2. 是否进入 Phase 2（竞品分析+设计哲学创建）？（是 / 否）
```

---

## 关键提示

- 品牌名确认后**立即**创建 brand-profile.md，但**只填 Layer 1 字段**
- 如果用户只要求做单张海报，Layer 2 聚焦海报相关信息，跳过品牌类问题
- 追问中如果用户提到行业关键词（如"有机""医疗""金融"），在 brand-profile.md 中标记为「合规重点关注项」，但不展开讨论（Phase 5 再处理）
- **严禁在 Phase 1 提前透露 Phase 2-6 的具体内容**（渐进式披露 PD-6）
